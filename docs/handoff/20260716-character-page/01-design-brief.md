# 角色分析頁翻新 — Claude Design 設計需求

> 2026-07-16。對應工程計畫 `docs/plans/20260716-character-page-revamp.md`(範圍 #0–#13)。
> 交付格式:依既有流程產出 `.dc.html` canvas,工程實作以 canvas 為準。

## 產品脈絡

StorySphere 是敘事文本分析平台。角色分析頁(`/books/:bookId/characters`)是**逐角色巡覽的檢視器**:左側角色清單 + 右側分析內容(3 primary tab:人物概覽/語音風格/認知狀態;概覽內 4 sub-tab:人格/行為/關係/弧線)。此結構為 2026-05-16 定案,**本次翻新不推翻**。

翻新主題:

1. **從靜態報告變成可導航的分析工具** — 現況所有分析結論(引言、證據、關係、事件)都是死文字,無法溯源回原文、無法跳到相關角色/事件
2. **進頁引導** — 未選角色時只有一行提示,不回答「這本書有哪些角色群像?誰重要?從誰開始?」

測試資料:《大唐雙龍傳》實驗書,**99 位角色(98 位未分析)/ 7 章**。設計請以這種「一位已分析 + 大量長尾」與「全部已分析」兩種極端都成立為準。

## 要設計的元件

### A. 角色總覽 landing — 最高優先

內容區「未選取角色」狀態升級為 cast overview(不開新路由)。三個層級:

1. **群像摘要條**:「99 位角色 · 已分析 1 · 未分析 98」+ 批次生成入口。批次入口需有**兩檔**:「先生成前 10 位要角」(依提及數)/「生成全部」,confirm 顯示對應數量。
2. **主要角色 spotlight**:依提及數降冪取前 4–6 位。每卡:hash 配色頭像(沿用左清單邏輯)+ 名稱(serif)+ 原型 badge + `profileSummary` 前兩行(資料見 `sample-payloads/characters-list.json` 的 `content` 欄)+ 提及量 meta + 狀態點。**第一位做加大的 hero card**;若該角色尚未分析,hero 改為「核心角色尚未分析 → 建立」CTA 變體。
3. **其他角色 — 按派系分組**:依 `sample-payloads/factions.json`(實測 10 派系:隋室、宋閥、主角團、宇文閥…)每派系一張橫列卡:代表 3 人 + 成員數 + 凝聚度,點開展開成員 row;無派系者入「其他」組。派系卡附「在圖譜中查看」深連結。**注意:實測 66/99 位無派系歸屬**——「其他」組是最大宗,必須預設收合(如顯示前 N 位 + 「展開其餘 M 位」),派系卡負責組織有意義的那三分之一。

另需設計**回程入口**:選中角色後可返回總覽(左清單頂部「← 角色總覽」或標題列麵包屑,擇一)。此區取代現有「快速前往已分析角色」列表。

> **2026-07-16 修訂(設計進行中回饋)**:canvas 已把層級二+三演化為**「提及量排行 / 定位象限」雙視圖**,經工程可行性查核採納,取代上述 spotlight 卡片網格與派系分組清單:
> - **排行視圖**:hero(排名第 1,未分析時 CTA「建立核心角色分析」)+ 排行列(提及量 bar 線性尺度)。長尾收合規則仍適用(99 列不平鋪)
> - **象限視圖**:X=敘事存在感(log)、Y=結構重要性(PageRank/中介中心性,後端將新增小端點)、泡泡大小=關係連結數、顏色=派系。三個資料現實請納入設計:(a) 切線建議取兩軸中位數,否則角色擠單側;(b) 99 顆泡泡 label 只標 top N、其餘 hover;(c) **66/99 位無派系** → 大多數泡泡同色,無歸屬者建議低飽和/描邊處理
> - 派系從分組結構改為象限顏色編碼;A-3 的派系分組清單**作廢**,批次兩檔入口(A-1)與回程入口需求**不變**

### B. Arc 弧線時間軸

現況是純文字卡直列(每階段:章節區間 + phase 名 + 描述)。改為章節軸上鋪 phase 區段 + keyEvents marker,文字描述保留在軸下方(點擊區段捲至描述)。**實測資料的區間會重疊**(Ch.1-2 / Ch.2-3 / Ch.3-5,相鄰段共享邊界章),設計稿需含此 case。認知 tab 已有一條可拖曳的章節軸元件(全寬 axis + ticks + 事件 marker + 游標),兩者的軸線語言可統一設計。

### C. 認知對照視圖 — 雙角色資訊差

認知 tab 新增「對照另一角色」:選第二位角色後,三欄呈現「只有 A 知道 / 都知道 / 只有 B 知道」,**共用同一條章節游標**,拖曳同步更新。互動模式(非視覺)可沿用既有框架對照 drawer 的慣例:右側抽屜覆蓋內容區、Esc/backdrop 關閉。第二角色選擇器只列有資料的角色。事件量級見 `sample-payloads/epistemic-state-ch7.json`(單角色已知 28 / 未知 34)。

### D. 溯源與跨頁導航 affordances

分析內容中的「可點性」線索,涵蓋四處:

1. **引言/證據可點** → 跳閱讀頁對應段落:PersonaPane 證據列、關係頁引言、語音頁代表引言。需要 hover/點擊 affordance,且與純文字段落有區別但不搶閱讀
2. **關係 target 分組卡**:資料中同一對象會出現多段關係(實測:楊廣對宇文化及同時是「下屬」+「敵人」,見 `sample-payloads/entity-analysis.json` 的 `cep.relations`)→ 按對象分組,一個對象一張卡、多段關係堆疊;對象若是可跳轉的角色,需可點樣式
3. **認知事件列可點** → 跳閱讀頁;timeline **同章多 marker 聚合成帶數字的點**(現況十多顆點疊在同一位置),hover 展開清單
4. **行為頁 keyEvents** → 連到事件分析頁的連結樣式

### E. 左清單升級

- 每列加**提及量迷你 bar**(相對全書最大值;寇仲滿格、龍套一絲)
- 排序後長尾自然沉底;搜尋涵蓋原型名(無需新視覺,標注即可)
- 未分析 item 的「生成分析」按鈕與 bar 的共存排版

### F. 生成中狀態(低優先)

單角色生成中改為階段 checklist(彙整證據 → 原型判定 → 弧線 → …)。**沿用符號意象頁 `InterpretationGenerating` 的既有模式即可**,只需確認在本頁 content area 的擺放,不必重新設計。
