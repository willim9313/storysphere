# 閱讀頁翻新 — Claude Design 設計需求

> 2026-07-13。對應工程計畫 `docs/plans/20260713-reader-page-revamp.md`（Batch 1–3）。
> 交付格式：依既有流程產出 `.dc.html` canvas，工程實作以 canvas 為準。

## 產品脈絡

StorySphere 是敘事文本分析平台。閱讀頁（`/books/:bookId`）定位是**檢視器（inspector）**，不是沉浸式閱讀器：正文以自動 chunking 的段落卡片呈現（含 `#N` 編號、實體 chips、關鍵字），這個結構是定案，設計不要試圖把它改成連續文流。

本次翻新目標：
1. 把「看起來可互動但點不動」的元素變成真正的入口（行內實體標註、entity chips、認知狀態事件列表）
2. 補上檢視動線（章節間導航、進度、專注模式）

測試資料：《大唐雙龍傳》實驗書，7 章 / 58 chunks / 202 實體，實體標註密度很高（單一 chunk 常有 10+ 個標註）——設計時請以這種高密度場景為準，不要只考慮稀疏情況。

## 現有版面

```
[全站側欄] [欄1 書籍總覽 232px 可收合] [欄2 章節列表 224px 可收合] [欄3 chunk 內容 flex] [認知狀態側欄 280px 開關式]
```

- 欄 3 有 sticky header：章節標題 + chunk 數（左）、標註密度三段開關「全部/角色/關」+「角色視角」按鈕（右）
- 欄間有手繪感 Bezier 曲線連接選中章節卡與內容區
- 行內實體標註（`.entity-mark`）：預設僅該型別色細底線，hover 浮淡色塊——這是既有設計語言（UI_SPEC §1.4），新互動要建立在它之上
- 窄螢幕 ≤768px：欄 1/2 自動收合成 36px icon 條

現況截圖（提供給設計參考）：初始畫面、章節展開、標註過濾、認知狀態面板、捲動後、雙欄收合、ink 主題各一張。

## 設計系統

**視覺樣式將套用新的 design system（用戶自備），本文件不約束視覺語言**；下列現況資訊僅供理解語意結構與功能約束：

- 實體分**七個型別**（character / location / organization / object / concept / event / other），每型別有獨立識別色——新系統需保留「型別可由顏色區分」的語意，具體色值由新系統決定
- 現有兩種呈現載體：chips（`.pill`，清單用）與行內標註（`.entity-mark`，正文用，現況為細底線 + hover 浮色塊）——新設計可重定義樣式，但「正文標註不搶字流」的原則建議保留
- 工程實作面仍會走 CSS token 架構（色值集中管理、支援多主題），新系統的產出需能收斂成 token
- 文案需 zh-TW / en 雙語（i18n）

## 要設計的元件

### A. 實體卡（Entity Card）— 最高優先

**觸發**：點擊行內實體標註或 chunk 頂部 entity chip。

**可用資料**（API 已存在，欄位固定）：
- 實體名稱、型別（七類之一）
- 全書出現次數 `total` 與出現段落列表：每項有章節標題/編號、chunk 編號、段落內文（可截摘錄）——列表可能長達數十項，需可捲動
- 若為角色且已生成深度分析：`profileSummary`（一段文字）、原型 archetypes；未生成時 API 回 404 → 顯示「未生成」狀態（不需在卡內提供生成按鈕）

**動作**：
1. 點出現段落項 → 跳轉到該段落（同章捲動定位、跨章切章後定位，既有 flash 高亮）
2. 「角色分析」→ 前往角色分析頁（僅 character 型別顯示）
3. 「在圖譜中查看」→ 前往知識圖譜頁聚焦該節點

**需要設計決定的**：
- 形式：anchored popover、浮動卡、或右側面板？考量點——欄 3 寬度會隨欄 1/2 收合大幅變動；認知狀態側欄可能同時開著；窄螢幕要能用
- 高密度標註下的觸發精度（行內標註是文字級元素，間距很小）
- loading 狀態（段落列表是點擊後才 fetch）
- 關閉方式（點外部 / Esc / 關閉鈕）

**行為約束**：標註密度開關切到「關」時行內標註不可點；「角色」模式下非角色標註不可點（與視覺一致——看不見的東西不該可點）。

### B. 認知狀態面板 — 事件列表可點

現況：已知事件（綠）/ 未知事件（黃）/ 誤信（紅）三組純文字列表項，每項已標 `Ch.N`。

需要：可點的 affordance + hover 狀態，點擊跳轉到對應章節位置。保持現有紅黃綠色彩語意（走 `--color-{success|warning|error}` token）。順帶可整理項目的視覺層級（事件標題 vs 章節標籤）。

### C. 章節導航 + 進度指示

- 章末「← 上一章 / 下一章 →」（第一章/最末章時單邊）
- chunk 區浮動「回頂部」鈕（捲動一段距離後才出現）
- sticky header 增加「第 N / M 章」+ 章內捲動進度細條
- 注意 sticky header 右側已有三段開關 + 角色視角按鈕，空間有限，新增元素要收得進去（含窄螢幕）

### D. 專注模式 + 排版控制

- sticky header 工具列加「專注模式」開關：一鍵收合欄 1+欄 2（現在要分別按兩次），正文加 max-width 上限置中
- 字級調整 2–3 檔（設定持久化）
- **保留 chunk 卡片結構**——專注模式只是變寬變乾淨，不改變卡片呈現

## 不要動的部分

- chunk 卡片結構（`#N`、chips、keywords 的資訊架構）
- Bezier connectors
- 章節卡手風琴行為、欄 1/2 的既有收合機制（專注模式是在其上加一鍵入口）
- 實體色跨主題不覆寫是既有設計 contract；若設計認為 ink 主題下該調整，另開議題討論，不在本次範圍
