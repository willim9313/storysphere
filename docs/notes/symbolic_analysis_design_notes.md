## 一、背景與定位

### 符號學在整體分析框架中的位置

本模組屬於「反覆出現的模式」分析路線的一部分，與原型批評並行但互補：

| 框架 | 問題 | 移動方向 | 系統化難度 |
|------|------|----------|------------|
| **符號學** | 這個符號在這個文本裡是什麼意思？ | 在文本內部深挖 | 高（詮釋開放） |
| **原型批評** | 這個模式從哪裡來？為什麼有共鳴？ | 從文本向外連結 | 中（模式匹配） |

**兩者缺一不可**：
- 符號學建立文本內部的意義系統
- 原型批評解釋這個意義系統的文化根源和共鳴來源

### 核心設計原則

> 符號學的詮釋自由度來自「追蹤什麼」的選擇，但詮釋本身必須被文本內部證據支持。

這個原則決定了系統設計的邊界：
- 系統負責**觀察**（統計、分佈、關係）
- 人或 LLM 負責**詮釋**（意義、命題）

## 二、核心概念釐清

### 什麼是「符號」

符號學追蹤的不是任意詞彙，而是在文本中**承載意義的意象**。

- 能指（signifier）：文本中出現的具體元素，如「光」、「鏡子」、「門」
- 所指（signified）：這個元素在文本中指向的意義

**重要區分**：
- 人物實體（張三、李四）→ 已由現有 KG 模組處理
- 意象實體（光、暗、水、手）→ 符號學模組的新需求

### 符號分析的操作定義

一個符號分析的完整流程包含三個步驟：

1. **建立出現清單**：記錄每次出現的位置、語境、周圍詞彙
2. **歸納語義場**：從清單中歸納符號的意義分佈和變化
3. **提出詮釋**：語義場在說什麼命題？

系統可自動化第 1、2 步，第 3 步由人或 LLM 輔助完成。

## 三、功能架構（三層設計）

### 第一層：候選符號發現

**目標**：回答「有什麼值得追蹤？」

**輸入**：全書文本（已分章節、已分段）

**功能**：
- 高頻意象提取：去除停用詞後，識別出現頻率異常高的名詞/意象
- 情感密度標記：識別在情感強烈段落（衝突、死亡、轉折）密集出現的意象
- 分佈不均檢測：識別在特定章節突然大量出現或消失的意象

**輸出**：候選符號清單，附帶：
- 出現頻率
- 章節分佈圖
- 情感密度分數

**備註**：此層為純統計工作，不涉及語義詮釋。

---

### 第二層：共現網絡建構

**目標**：回答「這些符號之間有什麼關係？」

**輸入**：候選符號清單 + 全書文本

**功能**：
- 共現矩陣：計算哪些意象總是一起出現、哪些從不同時出現
- 語境採樣：每個符號出現時，提取周圍的語義場（滑動窗口）
- 語境變化追蹤：同一符號在不同章節的語境是否有系統性差異

**輸出**：符號關係網絡，可視覺化為圖結構

**與現有架構的連接**：
- 資料結構與知識圖譜相同，節點類型不同（意象實體 vs 人物/事件實體）
- 可考慮整合進現有 KG，或作為平行圖層

---

### 第三層：詮釋輔助介面

**目標**：將前兩層結果組織成對分析者有用的格式

**輸入**：符號清單 + 共現網絡 + 語境採樣結果

**功能**：
- 符號時間軸：某符號的所有出現實例，按章節順序排列，附帶前後文
- 語境變化節點標記：自動標記語境發生明顯變化的位置
- 符號配對呈現：共現關係強的符號組，自動配對展示出現實例

**輸出**：供分析者閱讀和詮釋的結構化報告

**設計原則**：
- 系統提供「光在摩瑞亞章節出現頻率比其他章節低 70%，且主要與『消失』、『恐懼』共現」
- 系統**不提供**「光代表希望」
- 這條線不能混淆

## 四、最大技術挑戰：意象實體識別

### 問題描述

符號學的新需求是識別**意象實體**，這和現有的人物/地點實體識別有根本差異：

| | 人物/地點實體 | 意象實體 |
|---|---|---|
| **性質** | 具體、固定 | 抽象、語境依賴 |
| **識別方式** | NER（命名實體識別） | 需要語義理解 |
| **邊界** | 清楚 | 模糊（「光」vs「火」是同一符號嗎？） |
| **複雜度** | 低 | 高 |

### 已知困難

1. **同義詞聚合**：「光」、「火焰」、「燭光」、「星光」是否屬於同一符號族群？
2. **語境依賴**：同一個詞在不同語境可能承載完全不同的符號功能
3. **抽象層次**：意象可能以隱喻形式出現，不是直接的詞彙匹配

### 研究方向（待探索）

- 詞嵌入聚類：用語義相似度將相關詞彙聚合成符號族群
- LLM 輔助標注：用 LLM 識別段落中具有符號功能的意象
- 人工種子 + 擴展：分析者提供初始符號（如「光」），系統自動擴展到相關詞彙

> ⚠️ **這個問題需要獨立的專項研究，是整個符號學模組的前置依賴。對應 B-017。**

## 五、與現有 StorySphere 架構的整合

### 可複用的現有能力

| 現有能力 | 符號學用途 |
|----------|------------|
| 文本分塊（Chunker） | 提供章節/段落邊界，用於分佈分析 |
| 向量搜索（Qdrant） | 找語義相似的出現實例 |
| 知識圖譜（NetworkX） | 共現網絡可用相同資料結構 |
| 實體提取 Pipeline | 可擴展以支援意象實體識別 |

### 新增需求

- 意象實體類型（ImagerEntity）的 Domain Model（對應 B-018）
- 符號共現圖層（可作為 KG 的平行圖層）（對應 B-020）
- 語境採樣工具（滑動窗口提取）（對應 B-019）
- 符號時間軸視覺化介面（對應 B-021）

## 六、開放問題（待研究）

1. **意象實體識別策略**：詞嵌入聚類 vs LLM 標注 vs 人工種子擴展，哪個最可行？（→ B-017）
2. **符號圖層與 KG 的關係**：整合進現有 KG 還是獨立圖層？（→ B-020 設計決策）
3. **詮釋的持久化**：分析者的詮釋結果如何儲存和版本管理？（→ B-022 範疇）
4. **多書比較**：符號學分析是否要支援跨書比較（同一符號在不同作品的意義差異）？（長期）
5. **量化指標**：如何評估符號識別的品質？（→ B-017 驗收條件）
